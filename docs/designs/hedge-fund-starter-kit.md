## Problem Statement

Enterprises need AI agents that connect to their company data, run governed (auditable, permissioned, safe), and don't require a $500K consulting engagement to deploy. Koi is an open-source, self-evolving agent engine where everything is file-based and auditable. Nexus is its virtual file system that seamlessly connects personal agents to company data.

Target: non-technical teams at small hedge funds (5-50 people) who need AI agents wired into their business data but have no engineering team.

## Vision

### 10x Check
Koi becomes the "Salesforce for AI agents" in financial services. Every hedge fund, family office, and boutique shop runs Koi. Vertical skill packs auto-configure for each firm type. Nexus connects to Bloomberg, email, CRM, compliance databases, and market data feeds. The community registry has firm-specific skills shared across the industry. ACE means every deployment gets smarter over time. Compliance officers trust it because session replay + governance presets handle SEC/FINRA requirements out of the box.

## Scope

### Core Starter Kit
- New "hedge-fund" preset + template in existing `@koi/starter` (NOT a new L3 package)
- Docker Compose for one-click deployment
- 2-3 pre-built skills (research, summarization, compliance audit — flag-only mode)
- Reuse existing `@koi/channel-email` for IMAP (no new IMAP connector)
- New Google Drive Nexus connector (L2 — genuinely doesn't exist)
- OAuth tokens encrypted at rest in Nexus

### Accepted Expansions (from CEO review)

| 3 | Guided data source wizard in TUI | M | Non-technical users connect data without YAML |

### Key Design Decisions (revised per eng review + Codex)
- **Compliance skill: flag-only mode.** Never declares "compliant." Flags concerns for human review.
- **Reuse channel-email for IMAP.** Existing `@koi/channel-email` handles real-time email via IMAP IDLE. Nexus email indexing deferred until concierge deployments validate the need.
- **New preset, not new package.** Hedge fund config lives as a preset/template in existing `@koi/starter`, following the demo/mesh/sqlite pattern. No new L3 surface area.
- **Compliance scoped to "audit trail + building blocks."** Full audit trail, configurable retention, encrypted credential storage. NOT claiming SEC/FINRA compliance — that requires retention policies, WORM storage, and controls that will be informed by actual deployment experience.
- **OAuth tokens encrypted in Nexus.** Key management details (rotation, revocation, backup) to be designed during implementation.
- **Session replay: cursor-based pagination.** Immutability, redaction boundaries, and export to be designed with input from first compliance officer encounter.

## UX Design (added per design review)

### First-Boot Experience
On `docker compose up` → `koi up`, the TUI auto-seeds demo data AND auto-runs a research task so the user sees a live agent working within 30 seconds. No blank screens. The user's first experience is watching an agent research "ACME Corp" using demo data.

### Information Hierarchy

**Data Source Wizard (step 1 of 3):**
- PRIMARY: Source type selector (Email, Google Drive, Slack)
- SECONDARY: Description of selected source
- TERTIARY: "What this connects to" explanation
- Footer: [Enter] Select  [Esc] Cancel  [?] Help

**Session Replay:**
- Header: Session name, agent, duration, event count
- PRIMARY: Chronological event timeline with tool calls + agent reasoning
- SECONDARY: Event detail panel (on Enter)
- TERTIARY: Metadata (duration, tokens, cost)
- Footer: [up/down] Navigate  [Enter] Detail  [f] Filter  [/] Search

**Health Check (`koi doctor`):**
- PRIMARY: Pass/fail per component with symbols (pass=green, fail=red, warn=yellow)
- SECONDARY: Actionable fix for each failure ("Run 'koi wizard' to reconnect email")
- TERTIARY: System metadata (uptime, version)

### Interaction States

| Feature | Loading | Empty | Error | Success | Partial |
|---------|---------|-------|-------|---------|---------|
| Data source wizard | "Connecting..." spinner | N/A | "Auth failed — try again or enter token manually" | "Connected! Found 2,431 files" | 1 of 3 sources connected |
| Session replay | "Loading session..." | "No sessions yet. Run an agent first to see activity." | "Audit sink unavailable — check koi doctor" | Full timeline with events | Loading more (pagination) |
| Compliance filter | N/A | "No flags in this session." | N/A | Flag list with severity | N/A |
| Health check | "Checking..." per item | N/A | Red items with fix instructions | "All systems healthy" | Mixed green/red |
| Demo seed | "Seeding demo data... 47%" | N/A | "Seed failed — try 'koi demo reset'" | "Demo data ready. Try: ask about ACME Corp" | N/A |
| GDrive OAuth | "Visit this URL and enter the code: ABCD-1234" | N/A | "Auth timed out. A) Retry B) Enter token manually" | "Google Drive connected!" | N/A |

### Design Decisions (from design review)
- **OAuth: device code flow.** Display URL + code in TUI. User opens browser, enters code. Works in Docker, SSH, and local environments.
- **Auto-run demo on first boot.** `koi up` with hedge-fund preset auto-seeds demo data and runs a visible research task so non-technical users see value in <30 seconds.
- **Session replay shows reasoning.** Not just "Tool: search_email" but "Searching for ACME Corp mentions in last 30 days..." — makes it feel like watching an assistant, not reading a log.
- **Terminal constraints.** Minimum 80 columns. All actions keyboard-reachable. Status symbols work in monochrome (text + symbol, not color-only). Long content wraps, never truncates.
