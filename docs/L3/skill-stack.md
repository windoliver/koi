# @koi/skill-stack — Skill Composition & Hot-Plug Bundle

Layer 3 meta-package that composes skill gating, progressive loading,
auto-promotion middleware, file watching, and forge bridging into a single
`createSkillStack()` call. Enables agents to discover, load, hot-swap, and
security-gate skills at runtime without restart.

## What This Enables

**On-the-fly skill management for long-running agents.** Instead of manually
wiring 5 separate concerns (gating, provider, activator, watcher, forge bridge),
callers get:

- **Requirements gating** — skills filtered by platform, binary availability,
  and environment variables before mounting. A macOS-only skill is silently
  skipped on Linux.
- **Progressive loading** — skills load at metadata level (cheap) and auto-promote
  to body level (full content) only when referenced in model messages, minimizing
  context window usage.
- **Hot-mount / hot-unmount** — add or remove skills mid-session via `mount()` and
  `unmount()` without restarting the agent. Mount loads at body level and runs the
  security scanner.
- **File watcher** — watches skill directories for added/changed/removed SKILL.md
  files, triggering automatic mount/unmount. Enables drop-a-folder skill deployment.
- **Forge bridge** — listens for `ComponentEvent` from a forge provider, automatically
  mounting skills generated by `forge_skill` during a session.
- **Security presets** — three severity thresholds (restrictive, standard, permissive)
  control which security findings block skill activation.

### Before (manual composition)

```typescript
import { createSkillComponentProvider, createSkillActivatorMiddleware, createSkillFileWatcher, gateSkills, clearSkillCacheEntry } from "@koi/skills";
import { severityAtOrAbove } from "@koi/validation";

const { eligible } = gateSkills(skills);
const provider = createSkillComponentProvider({
  skills: eligible,
  basePath,
  loadLevel: "body",
  onSecurityFinding: (name, findings) => {
    if (findings.some(f => severityAtOrAbove(f.severity, "HIGH"))) {
      console.warn(`Blocked: ${name}`);
    }
  },
});
const activator = createSkillActivatorMiddleware({ provider });
const watcher = createSkillFileWatcher({
  dirs: ["/path/to/skills"],
  onChange: (event) => {
    if (event.kind === "added") provider.mount?.(fsSkill(event.name, event.dirPath), basePath);
    if (event.kind === "removed") provider.unmount?.(event.name);
  },
});

const runtime = await createKoi({
  manifest, adapter,
  middleware: [activator],
  providers: [provider],
});
```

### After (skill-stack)

```typescript
import { createSkillStack } from "@koi/skill-stack";

const { provider, middleware, mount, unmount, dispose } = createSkillStack({
  skills: [fsSkill("code-review", "./skills/code-review")],
  basePath: "/project",
  preset: "standard",
  watch: true,
  overrideDirs: ["/project/skills"],
});

const runtime = await createKoi({
  manifest, adapter,
  middleware: [...middleware],
  providers: [provider],
});

// Hot-mount a new skill mid-session
await mount(fsSkill("test-gen", "./skills/test-gen"));

// Remove a skill
unmount("code-review");

// Cleanup
dispose();
```

## Quick Start

```typescript
import { createSkillStack } from "@koi/skill-stack";
import { createKoi } from "@koi/engine";
import { fsSkill } from "@koi/core";

// Minimal — single skill, standard preset
const stack = createSkillStack({
  skills: [fsSkill("code-review", "./skills/code-review")],
  basePath: process.cwd(),
});

const runtime = await createKoi({
  manifest: { name: "my-agent", version: "0.1.0", model: { name: "claude-sonnet-4-5" } },
  adapter,
  middleware: [...stack.middleware],
  providers: [stack.provider],
});

// With file watching — auto-mounts new skills dropped into the directory
const watched = createSkillStack({
  skills: [],
  basePath: process.cwd(),
  watch: true,
  overrideDirs: ["./skills"],
  preset: "permissive",
  onSecurityFinding: (name, findings) => {
    console.warn(`[${name}] ${findings.length} findings`);
  },
});
```

## Security Presets

| Preset | Severity Threshold | Watch Default | Use Case |
|--------|--------------------|---------------|----------|
| `restrictive` | Reject MEDIUM+ | false | Production, untrusted skills |
| `standard` | Reject HIGH+ | true | Development, curated skills |
| `permissive` | Reject CRITICAL only | true | Local dev, trusted sources |

Presets control the security scanner gate: skills with findings at or above the
threshold trigger a warning via `onSecurityFinding`. The preset also sets the
default for file watching (can be overridden via `watch: true/false`).

## Hot-Plug Lifecycle

```
mount(skill)
  ├── Validate: reject duplicates, non-filesystem sources
  ├── Resolve: path validation + security boundary check
  ├── Clear cache: force fresh load from disk
  ├── Load: parse SKILL.md at "body" level
  ├── Security scan: run skill-scanner, check preset threshold
  ├── Register: add to components/levels/resolvedPaths maps
  └── Notify: fire ComponentEvent("attached") to listeners

unmount(name)
  ├── Remove: from all internal maps (components, levels, etc.)
  ├── Clear cache: remove cached entry for the skill path
  ├── Track: add to "skipped" list with reason "unmounted"
  └── Notify: fire ComponentEvent("detached") to listeners
```

Mount operations are serialized via a promise chain — concurrent mounts execute
sequentially. Unmount runs synchronously when no mount is in-flight; otherwise
it queues behind the pending mount to prevent state races.

## File Watcher

When `watch: true` and `overrideDirs` is provided, the stack creates a recursive
filesystem watcher (one per directory) that:

1. Watches for SKILL.md file changes using `fs.watch({ recursive: true })`
2. Debounces rapid changes (default: 250ms, configurable via `watchDebounceMs`)
3. Rescans the directory to detect added/changed/removed skills
4. Auto-mounts new skills, auto-unmounts deleted skills

The watcher safely handles:
- Non-existent directories (skipped with debug log)
- Rapid dispose (won't leak file descriptors even if init is still running)
- Callback errors (caught, non-fatal)

## Forge Bridge

When `forgeProvider` is supplied, the stack subscribes to `ComponentEvent` from
the forge provider. Skills generated via `forge_skill` during a session emit an
"attached" event with key `brick:skill:<name>`, which triggers automatic mounting.

```typescript
const stack = createSkillStack({
  skills: [],
  basePath,
  forgeProvider: forgeComponentProvider, // from @koi/forge
});
```

## Return Shape

```typescript
interface SkillStackBundle {
  readonly provider: ProgressiveSkillProvider;
  readonly middleware: readonly KoiMiddleware[];
  readonly mount: (skill: SkillConfig) => Promise<Result<void, KoiError>>;
  readonly unmount: (name: string) => void;
  readonly dispose: () => void;
  readonly config: ResolvedSkillStackMeta;
}

interface ResolvedSkillStackMeta {
  readonly preset: SkillStackPreset;
  readonly skillCount: number;
  readonly watcherCount: number;
  readonly gatingEnabled: boolean;
}
```

## Architecture

```
@koi/skill-stack (L3)
  ├── types.ts          — SkillStackConfig, SkillStackBundle, presets
  ├── presets.ts         — restrictive / standard / permissive specs
  ├── skill-stack.ts     — createSkillStack() factory
  └── index.ts           — public API surface

Delegates to:
  @koi/skills (L2)       — provider, activator middleware, watcher, gating
  @koi/validation (L0u)  — severity comparison for security gate
  @koi/skill-scanner (L0u) — security finding types
  @koi/core (L0)         — SkillConfig, ComponentEvent, Result types
```

## Requirements Gating

Skills can declare requirements in their SKILL.md frontmatter:

```yaml
---
name: docker-deploy
description: Deploy via Docker
requires:
  bins: [docker, docker-compose]
  env: [DOCKER_HOST]
  platform: [linux, darwin]
---
```

`gateSkills()` checks each requirement at stack creation time:
- **bins**: verifies binary exists on PATH via `Bun.which()`
- **env**: verifies environment variable is set
- **platform**: verifies `process.platform` matches

Skills that fail gating are silently filtered out with a reason — they appear
in the provider's `skipped` list but don't block agent startup.
